## 硬件准备

### 必需设备

| 设备 | 作用 | 连接方式 |
|------|------|----------|
| 超级电容模块 | 储能 + 功率测量 | 电源线 |
| 超级电容控制板 | CAN 通信 | CAN1 (TX:0x061, RX:0x051) |
| J-Link 调试器 | Ozone 调试 | SWD |
| 底盘电机 ×4 | M3508 | CAN1 (ID:1-4) |

### 接线检查

```
电池 → 超级电容模块 → 超级电容控制板 → 底盘板
                              ↓
                         CAN1 总线
                              ↓
                      M3508 电机 ×4
```

## 软件配置

### 启用功率控制

修改 `power_controller.h:20-21`：

```c
#define POWER_CONTROLLER_ENABLE 1  // 启用功率控制
#define RLS_ENABLE 0               // 初期关闭 RLS，用固定参数
```

### 功率限制设置

修改 `chassis.c:776`：

```c
float manual_power_limit = 80.0f;  // 根据实际需求调整 (W)
```

## 调试流程

### 第一步：验证通信

1. 上电，进入 Ozone 调试
2. 检查 `power_ctrl->status.cap_online == 1`
3. 检查 `power_ctrl->status.energy_feedback` 有数据更新

**如果 cap_online = 0**：
- 检查 CAN 接线
- 检查超级电容控制板 CAN ID 配置
- 检查 `cap->can_ins->rx_len > 0`

### 第二步：能量环调试

**目标**：缓冲能量稳定在目标值附近

**观察变量**：
- `energy_feedback`：当前能量
- `power_upper` / `power_lower`：功率上下限

**调整参数** (`power_controller.h:29-30`)：
```c
#define POWER_PD_KP 50.0f  // 比例增益，增大→响应更快
#define POWER_PD_KD 0.2f   // 微分增益，增大→抑制振荡
```

**调参方法**：
1. 先调 KP：从小到大，直到能量能快速恢复
2. 再调 KD：如有振荡，增大 KD 抑制

### 第三步：功率分配调试

**目标**：功率超限时合理分配到各轮

**观察变量**：
- `limited_output[0-3]`：限制后的电机输出
- `wheel_current[0-3]`：限制前的电机输出

**调整参数** (`power_controller.h:33-35`)：
```c
#define ERROR_POWER_DISTRIBUTION_THRESHOLD 20.0f  // 误差分配上限
#define PROP_POWER_DISTRIBUTION_THRESHOLD 15.0f   // 比例分配下限
```

### 第四步：RLS 参数辨识（可选）

**启用条件**：能量环稳定后

**启用方法**：
```c
#define RLS_ENABLE 1
```

**观察变量**：
- `power_ctrl->status.k1`：转速损耗系数
- `power_ctrl->status.k2`：力矩损耗系数

**正常范围**：
- k1: 0.1 ~ 0.5
- k2: 0.5 ~ 2.0

**异常处理**：
- 如果 k1/k2 发散（>10 或 <0），检查电机反馈数据
- 如果 k1/k2 不收敛，调整 `rls_lambda`（0.999~0.9999）

## 参数说明

### 功率模型参数

| 参数 | 默认值 | 说明 | 调整建议 |
|------|--------|------|----------|
| `k1_init` | 0.22 | 转速损耗系数 | 根据 RLS 结果调整 |
| `k2_init` | 1.2 | 力矩损耗系数 | 根据 RLS 结果调整 |
| `k3` | 2.78 | 静态功率损耗 | 一般不需调整 |
| `rls_lambda` | 0.9999 | RLS 遗忘因子 | 越大越稳定，越小越敏感 |

### 能量环参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `POWER_PD_KP` | 50.0 | 比例增益 |
| `POWER_PD_KD` | 0.2 | 微分增益 |
| `REFEREE_FULL_BUFF` | 60.0 | 满能量阈值 (J) |
| `REFEREE_BASE_BUFF` | 50.0 | 基础能量阈值 (J) |

### 电容参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `CAP_FULL_BUFF` | 230.0 | 电容满能量阈值 |
| `CAP_BASE_BUFF` | 30.0 | 电容基础能量阈值 |
| `MAX_CAP_POWER_OUT` | 300.0 | 电容最大输出功率 (W) |

## 常见问题

### Q1: 功率限制不生效

**检查项**：
1. `POWER_CONTROLLER_ENABLE == 1`
2. `cap_online == 1`
3. `energy_feedback` 有数据

### Q2: 电机输出被过度限制

**可能原因**：
- `energy_feedback` 过低
- `manual_power_limit` 设置过小
- PD 参数过激进

**解决方法**：
- 检查超级电容充电状态
- 增大 `manual_power_limit`
- 减小 `POWER_PD_KP`

### Q3: RLS 参数发散

**可能原因**：
- 电机反馈数据异常
- 功率测量噪声大

**解决方法**：
- 检查电机在线状态
- 增大 `rls_lambda`（如 0.99999）
- 暂时关闭 RLS，使用固定参数

### Q4: 能量环振荡

**现象**：`power_upper`/`power_lower` 剧烈波动

**解决方法**：
- 增大 `POWER_PD_KD`
- 减小 `POWER_PD_KP`
