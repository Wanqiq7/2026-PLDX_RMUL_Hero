📋 Vision/Hero_Vision YAML 参数完整指南

一、基础配置参数
参数: enemy_color
类型: string
示例值: "red" / "blue"
说明: 敌方颜色，决定过滤哪种颜色的装甲板
────────────────────────────────────────
参数: yolo_name
类型: string
示例值: yolov5 / yolov8 / yolo11
说明: 选择 YOLO 模型版本
────────────────────────────────────────
参数: device
类型: string
示例值: CPU / GPU
说明: 推理设备
────────────────────────────────────────
参数: use_traditional
类型: bool
示例值: true / false
说明: 是否使用传统视觉方法
调整建议：

- enemy_color：比赛前必须确认，设错会导致完全无法识别目标
- yolo_name：yolo11 精度最高但速度较慢，yolov5 速度快但精度 稍低
- 英雄机器人建议使用 yolo11，因为英雄射速慢，精度比帧率更重 要

二、相机参数
参数: camera_name
类型: string
示例值: hikrobot / mindvision
说明: 相机类型
────────────────────────────────────────
参数: exposure_ms
类型: double
示例值: 0.8
说明: 曝光时间（毫秒）
────────────────────────────────────────
参数: gain
类型: double
示例值: 18.0
说明: 增益
────────────────────────────────────────
参数: vid_pid
类型: string
示例值: "2bdf:0001"
说明: USB 设备 ID
调整建议与现象对照：
现象: 图像过暗，灯条不清晰
原因: 曝光不足
调整方法: 增大exposure_ms（如 0.8→1.5）
────────────────────────────────────────
现象: 图像过亮，灯条过曝糊成一片
原因: 曝光过度
调整方法: 减小 exposure_ms（如 2.0→0.8）
────────────────────────────────────────
现象: 图像噪点多
原因: 增益过高
调整方法: 减小 gain（如 18→12）
────────────────────────────────────────
现象: 暗处目标识别不到
原因: 增益不足
调整方法: 增大 gain（如 10→18）
────────────────────────────────────────
现象: 运动模糊严重
原因: 曝光时间过长
调整方法: 减小 exposure_ms（如 2.0→0.5）
英雄机器人特点：

- 英雄弹速较慢（约 16m/s），目标相对运动较慢，可适当增加曝光
- 建议 exposure_ms: 0.8~1.5，gain: 15~20

三、传统检测参数
参数: threshold
类型: int
示例值: 150
说明: 二值化阈值
────────────────────────────────────────
参数: max_angle_error
类型: double
示例值: 45
说明: 灯条最大倾斜角度（度）
────────────────────────────────────────
参数: min_lightbar_ratio
类型: double
示例值: 1.5
说明: 灯条最小长宽比
────────────────────────────────────────
参数: max_lightbar_ratio
类型: double
示例值: 20
说明: 灯条最大长宽比
────────────────────────────────────────
参数: min_lightbar_length
类型: double
示例值: 8
说明: 灯条最小长度（像素）
────────────────────────────────────────
参数: min_armor_ratio
类型: double
示例值: 1
说明: 装甲板最小长宽比
────────────────────────────────────────
参数: max_armor_ratio
类型: double
示例值: 5
说明: 装甲板最大长宽比
────────────────────────────────────────
参数: max_side_ratio
类型: double
示例值: 1.5
说明: 装甲板两侧灯条最大比值
────────────────────────────────────────
参数: max_rectangular_error
类型: double
示例值: 25
说明: 装甲板最大矩形误差（度）
────────────────────────────────────────
参数: min_confidence
类型: double
示例值: 0.8
说明: 分类器最小置信度
调整建议与现象对照：
现象: 灯条检测不到
原因: 阈值过高
调整方法: 降低 threshold（如 150→120）
────────────────────────────────────────
现象: 误检测到环境光源
原因: 阈值过低
调整方法: 提高 threshold（如 120→160）
────────────────────────────────────────
现象: 远距离装甲板丢失
原因: 灯条长度阈值过高
调整方法: 降低 min_lightbar_length（如 8→5）
────────────────────────────────────────
现象: 近距离误检测
原因: 灯条比例范围过宽
调整方法: 收紧 min/max_lightbar_ratio
────────────────────────────────────────
现象: 倾斜装甲板丢失
原因: 角度限制过严
调整方法: 增大 max_angle_error（如 45→55）
────────────────────────────────────────
现象: 大装甲板识别为小装甲板
原因: 比例范围不对
调整方法: 调整 min/max_armor_ratio
英雄机器人特点：

- 英雄主要打击英雄（大装甲板）和步兵（小装甲板）
- 建议 min_armor_ratio: 1.0，max_armor_ratio: 5.0覆盖两种装 甲板

四、Tracker（跟踪器）参数
参数: min_detect_count
类型: int
示例值: 5
说明: 从detecting 进入 tracking 所需的连续检测次数
────────────────────────────────────────
参数: max_temp_lost_count
类型: int
示例值: 25
说明: 临时丢失后保持跟踪的最大帧数
────────────────────────────────────────
参数: outpost_max_temp_lost_count
类型: int
示例值: 75
说明: 前哨站专用的临时丢失帧数
调整建议与现象对照：
现象: 目标频繁丢失重新锁定
原因: max_temp_lost_count 过小
调整方法: 增大（如 15→30）
────────────────────────────────────────
现象: 锁定错误目标后切换慢
原因: max_temp_lost_count 过大
调整方法: 减小（如 30→15）
────────────────────────────────────────
现象: 目标刚出现就开火，精度差
原因: min_detect_count 过小
调整方法: 增大（如 3→5）
────────────────────────────────────────
现象: 目标出现后反应慢
原因: min_detect_count 过大
调整方法: 减小（如 8→5）
────────────────────────────────────────
现象: 打前哨站时频繁丢失
原因: 前哨站旋转导致遮挡
调整方法: 增大outpost_max_temp_lost_count
英雄机器人特点：

- 英雄弹丸伤害高（100HP），宁可少打也不能打偏
- 建议 min_detect_count: 5~8，确保锁定稳定后再开火
- max_temp_lost_count: 20~30，避免频繁切换目标

五、Aimer（瞄准器）参数
参数: yaw_offset
类型: double
示例值: -0.8
说明: yaw 角度偏移（度）
────────────────────────────────────────
参数: pitch_offset
类型: double
示例值: -1
说明: pitch 角度偏移（度）
────────────────────────────────────────
参数: comming_angle
类型: double
示例值: 60
说明: 小陀螺时"来向"装甲板的角度阈值（度）
────────────────────────────────────────
参数: leaving_angle
类型: double
示例值: 20
说明: 小陀螺时"离开"装甲板的角度阈值（度）
────────────────────────────────────────
参数: left_yaw_offset
类型: double
示例值: -1
说明: 左侧发射管yaw 偏移（度）
────────────────────────────────────────
参数: right_yaw_offset
类型: double
示例值: -0.6
说明: 右侧发射管 yaw 偏移（度）
────────────────────────────────────────
参数: decision_speed
类型: double
示例值: 10
说明: 高/低速延迟切换阈值（rad/s）
────────────────────────────────────────
参数: high_speed_delay_time
类型: double
示例值: 0.026
说明: 高速旋转时的发弹延迟（秒）
────────────────────────────────────────
参数: low_speed_delay_time
类型: double
示例值: 0.010
说明: 低速/静止时的发弹延迟（秒）
调整建议与现象对照：
现象: 弹丸整体偏左
原因: yaw 偏移不对
调整方法: 增大 yaw_offset（如 -0.8→-0.3）
────────────────────────────────────────
现象: 弹丸整体偏右
原因: yaw 偏移不对
调整方法: 减小 yaw_offset（如 -0.8→-1.3）
────────────────────────────────────────
现象: 弹丸整体偏高
原因: pitch 偏移不对
调整方法: 减小 pitch_offset（如 -1→-2）
────────────────────────────────────────
现象: 弹丸整体偏低
原因: pitch 偏移不对
调整方法: 增大 pitch_offset（如 -1→0）
────────────────────────────────────────
现象: 打小陀螺时总打到装甲板边缘
原因: 来向/离开角度不对
调整方法: 调整 comming_angle/leaving_angle
────────────────────────────────────────
现象: 高速旋转目标打不中
原因: 延迟补偿不足
调整方法: 增大 high_speed_delay_time
────────────────────────────────────────
现象: 静止目标打不中
原因: 延迟补偿过大
调整方法: 减小 low_speed_delay_time
英雄机器人特点：

- 英雄通常有双发射管，需要分别校准 left_yaw_offset 和 right_yaw_offset
- 英雄弹速慢（~16m/s），飞行时间长，延迟补偿更重要
- 建议 high_speed_delay_time: 0.03~0.05

六、Shooter（射击控制）参数
参数: first_tolerance
类型: double
示例值: 5
说明: 近距离射击容差（度）
────────────────────────────────────────
参数: second_tolerance
类型: double
示例值: 2
说明: 远距离射击容差（度）
────────────────────────────────────────
参数: judge_distance
类型: double
示例值: 3
说明: 近/远距离判断阈值（米）
────────────────────────────────────────
参数: auto_fire
类型: bool
示例值: false
说明: 是否由自瞄控制射击
调整建议与现象对照：
现象: 近距离不开火
原因: 容差过小
调整方法: 增大 first_tolerance（如 3→5）
────────────────────────────────────────
现象: 近距离打偏
原因: 容差过大
调整方法: 减小 first_tolerance（如 5→3）
────────────────────────────────────────
现象: 远距离不开火
原因: 容差过小
调整方法: 增大 second_tolerance（如 1→2）
────────────────────────────────────────
现象: 远距离打偏
原因: 容差过大
调整方法: 减小 second_tolerance（如 3→2）
────────────────────────────────────────
现象: 中距离表现异常
原因: 距离阈值不对
调整方法: 调整 judge_distance
英雄机器人特点：

- 英雄弹丸珍贵（热量限制），必须确保命中
- 建议 first_tolerance: 3~5，second_tolerance: 1.5~2.5
- auto_fire: false 更安全，由操作手控制射击时机

七、能量机关（Buff）参数
参数: model
类型: string
示例值: assets/yolo11_buff_int8.xml
说明: 能量机关检测模型路径
────────────────────────────────────────
参数: fire_gap_time
类型: double
示例值: 0.2
说明: 两次射击的最小间隔（秒）
────────────────────────────────────────
参数: predict_time
类型: double
示例值: 0.05
说明: 预测提前量（秒）
调整建议：
现象: 打能量机关总是打到前一个位置
原因: 预测不足
调整方法: 增大 predict_time（如 0.05→0.08）
────────────────────────────────────────
现象: 打能量机关总是打到后一个位置
原因: 预测过度
调整方法: 减小 predict_time（如 0.08→0.05）
────────────────────────────────────────
现象: 连续射击但只有第一发命中
原因: 射击间隔过短
调整方法: 增大 fire_gap_time（如 0.2→0.3）
英雄机器人特点：

- 英雄打能量机关效率低（弹速慢、热量限制）
- 通常不建议英雄打能量机关，但如果需要，建议 predict_time: 0.06~0.10

十、参数调整流程建议

1. 相机参数调整
   └─ 确保图像清晰、灯条可见、无过曝/欠曝
2. 传统检测参数调整（如使用）
   └─ 确保灯条和装甲板能被正确检测
3. Tracker参数调整
   └─ 确保目标跟踪稳定、不频繁丢失
4. Aimer 偏移量校准
   └─ 静态目标校准 yaw_offset/pitch_offset└─ 动态目标校准 delay_time
5. Shooter 容差调整
   └─ 根据实际命中率调整容差
