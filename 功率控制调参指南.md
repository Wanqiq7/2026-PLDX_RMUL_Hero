# 功率控制调参指南

> 参考港科大 RM2024 功率控制模块开源文档

---

## 功率模型

底盘功率模型公式：

$$P = \sum_{i=1}^{n}\tau_i\omega_i + k_1\sum_{i=1}^{n}|\omega_i| + k_2\sum_{i=1}^{n}\tau_i^2 + k_3$$

| 符号 | 含义 |
|------|------|
| $P$ | 底盘总消耗功率 |
| $\tau_i$ | 第i个电机实际输出力矩 |
| $\omega_i$ | 第i个电机实际转速 |
| $k_1$ | 转速损耗系数 |
| $k_2$ | 力矩平方损耗系数 |
| $k_3$ | 系统静态功率损耗 |

---

## 1. 功率模型参数 (k1, k2, k3)

**文件位置**: `power_controller.c:26-28` 和初始化配置

| 参数 | 当前值 | 含义 | 调整方法 |
|------|--------|------|----------|
| `k1` | 0.22f | 转速损耗系数 | RLS自动辨识，或手动标定 |
| `k2` | 1.2f | 力矩平方损耗系数 | RLS自动辨识，或手动标定 |
| `k3` | 2.78f | 静态功率损耗 | 失能电机后测量 |

### k3 标定方法

```
1. 失能所有底盘电机
2. 从裁判系统或电容读取底盘实时功率
3. 取一段时间的平均值即为 k3
```

### k1/k2 调整

- 启用 RLS (`RLS_ENABLE = 1`) 时系统自动辨识
- 手动调参时，观察预测功率与实测功率差异逐步调整
- M3508 电机参考值：k1 ≈ 0.2, k2 ≈ 1.0

## 2. 能量环 PD 控制器参数

**文件位置**: `power_controller.h:29-31`

```c
#define POWER_PD_KP 50.0f  // 比例增益
#define POWER_PD_KD 0.2f   // 微分增益
```

### 调整建议

| 现象 | 调整方向 |
|------|----------|
| 能量环响应慢，电容能量波动大 | 增大 Kp |
| 能量环震荡 | 减小 Kp 或减小 Kd |
| 超调严重 | 增大 Kd |

> **注意**: 港科大文档提到 D 项容易使能量环震荡，建议谨慎使用，可考虑只用 P 控制器。

## 3. 功率分配阈值

**文件位置**: `power_controller.h:33-35`

```c
#define ERROR_POWER_DISTRIBUTION_THRESHOLD 20.0f  // E_upper
#define PROP_POWER_DISTRIBUTION_THRESHOLD 15.0f   // E_lower
```

### 分配策略切换逻辑

| 条件 | 分配策略 |
|------|----------|
| `Σerror > 20` | 100% error分配（大P分配） |
| `Σerror < 15` | 100% 等比缩放 |
| `15 < Σerror < 20` | 线性插值混合 |

### 调整建议

| 现象 | 调整方向 |
|------|----------|
| 坡上抖动/不稳定 | 增大 `PROP_THRESHOLD`，让等比缩放更早介入 |
| 起步响应慢 | 减小 `ERROR_THRESHOLD`，让大P分配更激进 |
| 走不了直线 | 检查阈值是否过低，导致分配过于敏感 |

## 4. 电容/裁判系统能量阈值

**文件位置**: `power_controller.h:37-43`

```c
// 裁判系统缓冲能量阈值
#define REFEREE_FULL_BUFF 60.0f   // 满能量目标
#define REFEREE_BASE_BUFF 50.0f   // 基础能量目标

// 电容能量阈值 (电压值 0-255)
#define CAP_FULL_BUFF 230.0f      // 电容满能量目标 (~90%)
#define CAP_BASE_BUFF 30.0f       // 电容最低能量目标
#define MAX_CAP_POWER_OUT 300.0f  // 电容最大输出功率
```

### 调整建议

| 参数 | 调整方向 |
|------|----------|
| `CAP_FULL_BUFF` | 设为电容最大电压的 85-90%，防止太满影响动能回收 |
| `CAP_BASE_BUFF` | 设为安全下限，低于此值强制削减功率 |
| `REFEREE_BASE_BUFF` | 可降低到 40-45J 增加安全裕度 |

## 5. 错误处理保守系数

**文件位置**: `power_controller.h:46-47`

```c
#define CAP_REFEREE_BOTH_GG_COE 0.85f  // 双断连时功率系数
#define REFEREE_GG_COE 0.95f           // 裁判系统断连系数
```

### 调整建议

- 双断连时仍超功率扣血 → 降低 `CAP_REFEREE_BOTH_GG_COE` 到 0.80f
- 系数越低越保守，但底盘性能也越差

## 6. RLS 参数

**文件位置**: `power_controller.c:319-325`

```c
RLS_Init_Config_s rls_config = {
    .lambda = config->rls_lambda,  // 遗忘因子
    .delta = 1e-5f,                // 协方差初始值
    .init_k1 = config->k1_init,
    .init_k2 = config->k2_init,
};
```

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `lambda` | 0.98~0.995 | 越大记忆越长，越小适应越快 |
| `delta` | 1e-5~1e-3 | 过大会导致首次更新参数剧变 |

## 调参流程

```
1. 标定 k3      → 失能电机，测量静态功率
2. 设置 k1/k2   → 参考 M3508: k1=0.2, k2=1.0
3. 启用 RLS     → 让系统自动辨识 k1/k2
4. 调整能量环   → 先只用 P 控制，稳定后再加 D
5. 调整分配阈值 → 根据坡上/起步表现微调
6. 调整保守系数 → 确保断连时不超功率
```
